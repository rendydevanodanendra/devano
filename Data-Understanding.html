Data Understanding dalam Data Mining#

Data Understanding adalah salah satu tahap penting dalam proses Knowledge Discovery in Databases (KDD) atau Data Mining. Ini merupakan langkah awal yang bertujuan untuk memahami data secara mendalam sebelum melakukan analisis lebih lanjut. Pemahaman data yang baik sangat krusial karena akan memengaruhi keberhasilan seluruh proses data mining.

Apa itu Data Understanding?#

Proses eksplorasi, investigasi, dan analisis awal terhadap dataset untuk:

  • Memahami struktur, isi, dan karakteristik data.

  • Mengidentifikasi pola, tren, anomali, atau masalah dalam data.

  • Menentukan apakah data tersebut sesuai dengan tujuan analisis atau perlu dilakukan preprocessing tambahan.

Tahap ini sering dianggap sebagai fondasi dari keseluruhan proses data mining karena hasilnya akan memandu langkah-langkah berikutnya, seperti data preparation, modeling, dan evaluation.

Tujuan Data Understanding#

Tujuan utama dari data understanding adalah:

  • Memahami Konteks Data:
    Mengetahui dari mana data berasal, apa arti setiap variabel, dan bagaimana hubungan antarvariabel.

  • Mendeteksi Masalah Data:
    Mengidentifikasi missing values, outliers, noise, atau inkonsistensi dalam data.

  • Menilai Kualitas Data:
    Memastikan bahwa data cukup berkualitas untuk digunakan dalam analisis. Hal ini meliputi pemeriksaan akurasi, kelengkapan, dan relevansi data.

  • Menentukan Arah Analisis:
    Berdasarkan pemahaman awal, peneliti dapat merumuskan hipotesis atau menentukan teknik data mining yang tepat.

Langkah-Langkah dalam Data Understanding#

Pengumpulan Data

  • Mengumpulkan data dari berbagai sumber, seperti database, file CSV, API, atau sensor.

  • Memastikan bahwa semua data yang relevan telah dikumpulkan sesuai tujuan analisis.

Sampling

  • Jika dataset sangat besar, sampling dapat digunakan untuk memahami subset data tanpa harus memproses seluruh dataset.

Profiling Data

  • Membuat ringkasan tentang setiap variabel dalam dataset, termasuk tipe data (numerik, kategorikal, ordinal), rentang nilai, dan frekuensi kemunculan.

  • Memahami hubungan antarvariabel menggunakan korelasi atau analisis asosiasi.

Visualisasi Data

  • Histogram: Untuk melihat distribusi data numerik.

  • Box Plot: Untuk mendeteksi outliers.

  • Scatter Plot: Untuk melihat hubungan antara dua variabel numerik.

  • Bar Chart/Pie Chart: Untuk memvisualisasikan proporsi variabel kategorikal.

Korelasi dan Asosiasi

  • Menghitung koefisien korelasi (misalnya Pearson, Spearman) untuk melihat hubungan linier antar variabel.

  • Menggunakan teknik asosiasi untuk menemukan pola hubungan antara variabel kategorikal.

Identifikasi Masalah Data

  • Missing Values: Menemukan data yang hilang atau tidak lengkap.

  • Outliers: Mendeteksi nilai-nilai ekstrem yang mungkin merupakan kesalahan atau anomali.

  • Duplikasi Data: Mengidentifikasi baris atau entri yang duplikat.

  • Inkonsistensi: Memeriksa apakah ada ketidaksesuaian dalam format atau nilai data.

Validasi Data

  • Memastikan bahwa data sesuai dengan domain atau konteks bisnis.

  • Memverifikasi apakah data mencerminkan realitas atau fenomena yang ingin dipelajari.

Contoh Kasus: Data Understanding dalam Praktik#

Misalkan Anda sedang menganalisis dataset penjualan produk di sebuah toko online. Berikut adalah contoh aktivitas data understanding:

  1. Eksplorasi Awal:

    • Dataset berisi kolom: ID_Produk, Nama_Produk, Harga, Jumlah_Terjual, Tanggal_Penjualan.

    • Menghitung statistik deskriptif untuk kolom numerik (Harga dan Jumlah_Terjual).

  2. Identifikasi Masalah:

    • Menemukan beberapa baris dengan nilai Harga = 0 (mungkin kesalahan input).

    • Ada tanggal penjualan yang tidak valid (misalnya, 31 Februari).

  3. Visualisasi:

    • Membuat histogram untuk melihat distribusi harga produk.

    • Membuat scatter plot antara Harga dan Jumlah_Terjual untuk melihat apakah ada hubungan.

  4. Profiling Data:

    • Kolom Nama_Produk memiliki 100 kategori unik.

    • Rata-rata harga produk adalah Rp500.000, dengan standar deviasi Rp200.000.

Manfaat Data Understanding#

  • Meningkatkan Kualitas Data: Dengan memahami masalah data, kita dapat melakukan preprocessing yang lebih efektif.

  • Menghemat Waktu dan Biaya: Menghindari kesalahan analisis yang disebabkan oleh data buruk.

  • Mendukung Pengambilan Keputusan: Memberikan wawasan awal yang berguna untuk merumuskan strategi analisis.

  • Mengurangi Risiko Kesalahan Model: Data yang dipahami dengan baik akan menghasilkan model prediktif yang lebih akurat.

Kesimpulan#

Data understanding adalah langkah kritis dalam data mining yang membantu memastikan bahwa data yang digunakan berkualitas dan relevan. Dengan melakukan eksplorasi dan analisis awal, kita dapat mengidentifikasi masalah, memahami pola, dan menentukan arah analisis yang tepat. Tanpa pemahaman data yang baik, hasil dari proses data mining mungkin tidak akurat atau bahkan menyesatkan.

Untuk memahami Data Understanding perlu library#

  1. Bahasa pemrograman misal Python (Pandas, Matplotlib, Seaborn) atau tools visualisasi seperti Tableau/Power BI.

  2. Mencatat semua temuan selama eksplorasi data untuk referensi di tahap selanjutnya.

  3. Libatkan domain expert jika data berkaitan dengan bidang spesifik (misalnya medis, keuangan).

1. Pengumpulan Data#

Langkah Pengumpulan Data#

Sebelum melakukan proses data mining, langkah pertama yang perlu dilakukan adalah pengumpulan data. Sebagai contoh, dalam studi ini kita menggunakan dataset Iris yang diambil dari dua sumber database berbeda di platform Aiven.io.

Apa itu Aiven.io?#

Aiven.io adalah platform layanan cloud terkelola yang menyediakan berbagai infrastruktur data open source secara siap pakai. Dengan Aiven, perusahaan dan pengembang dapat menggunakan database seperti PostgreSQL, MySQL, Apache Kafka, serta layanan lainnya (misalnya Elasticsearch, Redis, dan InfluxDB) tanpa harus repot mengelola infrastruktur dan skalabilitasnya sendiri. Aiven menangani aspek-aspek seperti keamanan, backup, monitoring, dan pemeliharaan, sehingga pengguna dapat fokus pada pengembangan aplikasi dan analisis data.

Dataset Iris#

Dataset Iris adalah dataset klasik yang sering digunakan dalam penelitian data mining dan machine learning. Dataset ini mengandung data mengenai tiga jenis bunga Iris:

  • Iris setosa

  • Iris versicolor

  • Iris virginica

Setiap sampel pada dataset ini memiliki empat fitur utama:

  • Panjang sepal

  • Lebar sepal

  • Panjang petal

  • Lebar petal

Sumber Data#

  • Data Iris Petal: Disimpan di database MySQL.

  • Data Iris Sepal: Disimpan di database PostgreSQL.

Dengan pengumpulan data dari kedua sumber tersebut, data mining dapat dilakukan untuk menggabungkan dan menganalisis informasi guna menemukan pola dan insight yang berguna.baik selanjutnya langsung ke proses pengumpulan data dan pengambungan data

2. Profiling Data#

Profiling Data adalah proses awal dalam data mining yang bertujuan untuk memahami dan mendokumentasikan karakteristik dasar suatu kumpulan data. Pada studi ini, kita menggunakan dataset Iris yang telah digabungkan dari dua sumber:

Data Iris Petal (dari MySQL)
Data Iris Sepal (dari PostgreSQL)

Dengan melakukan data profiling pada data gabungan tersebut, kita dapat mengidentifikasi kualitas data, pola-pola umum, serta potensi masalah seperti data hilang atau duplikasi. Berikut adalah beberapa aspek penting dalam data profiling:

1. Statistik Deskriptif
Menghitung nilai rata-rata (mean), median, modus, nilai minimum dan maksimum, serta variansi (variance) untuk setiap kolom numerik, seperti sepal_length, sepal_width, petal_length, dan petal_width.bertujuan untuk Memahami sebaran (distribution) dan karakteristik dasar dari tiap fitur. Misalnya, mencari panjang sepal rata-rata untuk spesies Iris tertentu.

2. Identifikasi Missing Values
Mendeteksi apakah terdapat kolom yang memiliki nilai hilang (missing) atau tidak lengkap, baik pada kolom petal maupun sepal.bertujuan untuk Menentukan apakah perlu dilakukan penanganan khusus seperti imputasi (mengisi nilai yang hilang) atau penghapusan data yang tidak lengkap. Hal ini penting untuk menjaga kualitas analisis selanjutnya.

3. Kualitas Data
Memeriksa konsistensi (misalnya, apakah panjang petal selalu lebih kecil dari panjang sepal), validitas (apakah ada nilai negatif untuk panjang atau lebar), dan keakuratan (apakah data sudah sesuai spesiesnya).bertujuan untuk Menjamin bahwa data yang akan digunakan untuk modeling memiliki kualitas memadai, sehingga hasil analisis atau model prediksi lebih dapat diandalkan.

4. Pemeriksaan Distribusi dan Outlier
Meninjau distribusi data pada tiap fitur (misalnya, menggunakan histogram atau box plot untuk petal_length dan sepal_length) guna mengidentifikasi outlier atau nilai yang tidak biasa.bertujuan untuk Menentukan apakah perlu melakukan transformasi data (misalnya, log transform) atau penanganan khusus pada outlier (misalnya, penghapusan atau pemisahan data ekstrem) agar model machine learning dapat bekerja lebih optimal.

5. Analisis Korelasi
Mengukur hubungan antara fitur-fitur (kolom) di dalam dataset gabungan, seperti sepal_length, sepal_width, petal_length, dan petal_width.bertujuan untuk Mengidentifikasi fitur-fitur yang saling terkait, yang dapat membantu dalam pembuatan model klasifikasi atau clustering. Misalnya, jika petal_length dan petal_width memiliki korelasi tinggi, hal ini dapat menjadi petunjuk untuk analisis lebih lanjut.

6. Profiling Metadata
Menganalisis informasi tentang tipe data (apakah numerik atau kategorikal), jumlah record total setelah penggabungan, dan atribut yang tersedia (misalnya, apakah ada kolom tambahan seperti class atau id).bertujuan untuk Mendapatkan gambaran umum mengenai struktur dataset. Misalnya, berapa banyak baris data setosa, versicolor, dan virginica, serta bagaimana distribusi jumlah record per spesies.

Dengan melakukan profiling data secara menyeluruh pada dataset Iris yang digabungkan dari MySQL (petal) dan PostgreSQL (sepal), kita dapat memperoleh pemahaman yang lebih baik mengenai kualitas data, distribusi fitur, dan hubungan antarvariabel. Informasi ini sangat membantu untuk menentukan langkah pembersihan (cleaning) dan persiapan data (data preparation) sebelum masuk ke tahap modeling.

3. Visualisasi Data#

Visualisasi data adalah teknik yang digunakan untuk menyajikan data dalam bentuk grafik atau diagram sehingga lebih mudah dipahami.
Berikut adalah beberapa jenis visualisasi yang sering digunakan:

Histogram
Digunakan untuk melihat distribusi data numerik.
Memungkinkan kita mengetahui apakah data berdistribusi normal, miring ke kiri/kanan, atau memiliki banyak puncak.
Sumbu X menunjukkan nilai variabel, sedangkan sumbu Y menunjukkan frekuensi kemunculannya.

Box Plot Membantu mendeteksi outliers (nilai ekstrem).
Menunjukkan distribusi data melalui lima statistik utama: minimum, kuartil pertama (Q1), median (Q2), kuartil ketiga (Q3), dan maksimum.
Outliers biasanya ditampilkan sebagai titik di luar whiskers.

Scatter Plot
Digunakan untuk melihat hubungan antara dua variabel numerik.
Jika ada pola tertentu (garis lurus, kurva, dll.), berarti ada hubungan antara kedua variabel.
Bisa digunakan untuk mendeteksi kluster atau outliers.

Bar Chart/Pie Chart
Bar Chart digunakan untuk memvisualisasikan data kategorikal (misalnya jumlah outlier vs non-outlier).
Pie Chart digunakan untuk menunjukkan proporsi kategori dalam bentuk lingkaran.

Histogram#

Histogram adalah grafik yang menunjukkan seberapa sering nilai tertentu muncul dalam data. Ini membantu kita memahami apakah data terdistribusi normal atau ada pola tertentu

sns.histplot(data[col], bins=20, kde=True, ax=axes[i//2, i%2]) → Membuat histogram untuk fitur tertentu dengan 20 bins (kelompok data) dan menambahkan kurva kepadatan (kde=True).
plt.tight_layout() → Supaya semua grafik tidak bertumpuk.
plt.show() → Untuk menampilkan grafiknya.

Fungsi Histogram:
Membantu melihat apakah data terdistribusi normal.
Bisa mendeteksi apakah ada outlier (misalnya, ada nilai yang jauh dari mayoritas data).

import numpy as np
from sklearn.neighbors import NearestNeighbors

# Hapus kolom "class"
combined_df = combined_df.drop(columns=["class"], errors="ignore")

# Inisialisasi model K-Nearest Neighbors
knn = NearestNeighbors(n_neighbors=5, metric="euclidean")
knn.fit(combined_df)

# Hitung rata-rata jarak ke k tetangga terdekat
distances_knn, _ = knn.kneighbors(combined_df)
mean_knn_distances = np.mean(distances_knn[:, 1:], axis=1)

# Tentukan threshold outlier
threshold_knn = np.mean(mean_knn_distances) + 2.5 * np.std(mean_knn_distances)

# Temukan indeks outlier
outliers_knn = np.where(mean_knn_distances > threshold_knn)[0]

combined_df["outlier"] = 0  # Default semua bukan outlier
combined_df.loc[outliers_knn, "outlier"] = 1  # Tandai outlier
